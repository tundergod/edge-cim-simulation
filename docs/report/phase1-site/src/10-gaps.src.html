缺口分類與 GO / NO-GO

這份 review 的結論：先把各單元的所有缺口攤開分三類——(a) Phase-2 blocker、(b) Phase-2 scope（已規劃）、(c) 可接受的 limitation——再給一個帶條件的 GO/NO-GO。分類是工程判斷，最終由專案負責人拍板；每一格都附「為何這樣歸類」。

§0結論摘要

(1) Phase 1 是否按其誠實標準交付＋驗證？ → 是。decode 主路徑（CIM G_eff + 記憶體 + GPU attention + CPU 支援 + 端到端 recompose 8B hold-out {{recompose.err_8b_pct}}%）達量測級可信，且 CIM 已 Card 重驗（median {{cim.card_median_pct}}% / p95 {{cim.card_p95_pct}}%，CARD_REVALIDATED）。完整、可換型號的 analytic 元件層齊備（M3/M6 整合層除外，屬 Phase 2）；重型 sim（Ramulator2 / ONNXim）以凍結介面 drop-in 就緒。所有非量測項都誠實標 simulated / assumption / borrowed。

(2) 進入 Phase 2 是否安全？ → 帶條件的 GO。沒有任何缺口會擋住「開始 Phase 2（實作 M3+M6 並整合）」；prefill / 大形狀的定量宣稱有一個量測前置條件附在其上（見 §2）。

§2GO / NO-GO 判定conditional GO

判定：元件層條件式 GO 進入 Phase 2。可以進入 Phase 2（實作 M3 事件引擎 + M6 排程器，整合成端到端模擬器）。理由：A 類為空——所有元件介面就緒、decode 主路徑量測級可信（recompose 8B hold-out {{recompose.err_8b_pct}}%、CIM Card 重驗 {{cim.card_median_pct}}%）、缺口皆已誠實標註且不阻止整合開工。

附帶條件（綁在特定宣稱上，非綁在開工上）：

conditional GO — 具名條件
條件綁在哪個宣稱
NPU 由 L4 端到端裁決無 silicon（#13）；Phase 2 整機若含 NPU 路徑，結果以 L4 端到端對照裁決，provenance 傳播 simulated 標籤。
thermal / 多單元競爭 = Phase-2 工作Aetina 送修期間並發 silicon 不可得；熱（Phase 0.4）與競爭驗證為 Phase 2 工作，板恢復後做。
prefill 端到端 TTFT 驗證元件層 prefill 已 Card-native 校準（M∈{2..{{cim.prefill_m_max}}}、cliff median {{cim.multitile_new_median_pct}}%）；剩整條 TTFT（attention + host overhead 組裝）屬 Phase 2 整合。
KV-append DRAM BW 隔離校準SPIKE INCONCLUSIVE；係數暫用 M2 {{kv.spike_m2_bw}} GB/s，Phase 2 板恢復後隔離校準。
混合精度成本基礎 = 非量測前置條件。conversion-op（CIM-INT8 × GPU-FP16 邊界的 dequant/requant）於 Phase 2 解析建模——memory-bound cast，由既有 M2/M4 per-op 模型 × M6 邊界穿越次數計價（ADR-0004 已於 2026-06-11 修訂）。精度固定於單元，故非量測缺口；Phase 2 把 cast op 插進 op stream 計價即可。

這份報告不保證模擬器完美；它保證我們精確知道自己站在哪、缺什麼、那些缺口會不會擋住 Phase 2。答案是：decode 地基穩固，整合可以開始；Phase 1.5 把 prefill / multi-tile / KV-BW 的元件級量測補齊（並修正了兩個過保守的編譯假設），剩 prefill TTFT 端到端組裝屬 Phase 2；混合精度成本則於 Phase 2 解析建模。
