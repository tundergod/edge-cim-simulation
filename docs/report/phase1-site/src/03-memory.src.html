記憶體單元

decode 是 memory-bound——速度由「把權重從 DRAM 搬出來」決定，不是算力。M2 把「搬資料」的成本算準：DRAM 串流頻寬（decode 牆）與 PCIe 每次呼叫的傳輸 floor，外加 KV-append 純頻寬 op。

§0本章角色

M2 模擬「搬位元組」的成本，兩個最基本功能：

① DRAM 串流——把權重 / KV 從 host DRAM 串出來，成本 = bytes / 有效頻寬。decode 每步要讀一遍整個權重，所以這是 decode 的記憶體牆。② PCIe / DMA 傳輸 floor——離散的 host↔device 搬移有一個固定的每次呼叫成本（floor），與搬多少位元組關係不大。

SRAM tier（L1/L2/D-IMC）歸在 M1-SPM，不在 M2 主路徑——8B 權重（~8 GB）遠大於 32 MiB L2，永遠塞不下、一定 spill 到 DRAM，所以 decode 的牆是 host LPDDR、不是 SRAM。M2 是有真矽錨點、也用了模擬的單元（LPDDR4x 量測 + Ramulator2 模擬），所以 simulated chip 亮著（和 M1 不同）。

§2Fit · 擬合equation · 逐參數講解
stream_us = nbytes / eff_BW_GBs # DRAM 串流（含 KV-append） transfer_us = floor_us + nbytes / pcie_BW_GBs # 離散 PCIe/DMA 搬移 kv_append_us = kv_bytes / eff_BW_GBs # 純頻寬 op

模型很簡單，關鍵在「有效頻寬」這個數字怎麼來。有效頻寬 = 理論峰值 × 效率；效率把 controller / NoC / 排隊等系統開銷都摺進去。三種 DRAM spec 各有自己的誠實標籤：
