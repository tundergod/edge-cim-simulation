就緒矩陣

一頁看完 Phase 1 所有單元的狀態：每個單元模擬什麼、資料源到哪一層（calibrated·fitted·simulated·assumption·borrowed）、驗證 gate 的頭條數字過不過、最大缺口在哪、能不能直接接 Phase 2。每個數字都從 committed JSON 以 {{key}} 佔位注入，沒有一個是手打的。

§0本章角色

這是全站的索引頁——後續每個單元頁（M1 CIM、M2 記憶體、M4-CPU、M4-GPU、M4-NPU、M5 工作負載、M7 能量／端到端）展開的細節，在此先濃縮成一列。誠實標籤（資料源層級）沿用各頁 honesty-chip 的決定：calibrated = 對我們的 Metis silicon 校準；fitted = 自家量測上的閉式擬合；simulated = 模型或外部 sim，未經本矽驗證；assumption = 規格／文獻推算；borrowed = 借自他人量測。一個單元常同時亮多個標籤，下表的「資料源層級」欄取該單元最具權威的那層當頭條，括號補次要層。

不在表內的兩塊：M3 事件引擎與 M6 排程器是 Phase 2 工作（Phase 1 只交 contract + 可調參數，未實作，設計使然非缺失）；M8 熱模型 pending（Metis Card 熱量測延到最後，Aetina 送修中）。

§2怎麼讀這張表

「資料源層級」欄取每個單元最具權威的誠實標籤當頭條，與各單元頁的 honesty-chip 一字對齊：例如 M4-NPU 的 calibrated 是關閉的（無 RKNPU2 silicon），所以它落在 simulated / borrowed；M7 的 calibrated 也關閉（無功耗遙測），落在 assumption。⚠ 標的是 on-but-caveated 的 assumption 層（橘色 chip）。

「Phase-2 就緒度」欄：✅ = 該單元按其誠實標準交付且通過該層驗證，介面可供 Phase 2 直接接上——注意「就緒」不等於「量測級精確」（如 LPDDR5、NPU ONNXim 是 simulated，但介面與標籤都誠實、可接，因此就緒）。⚠ = 有缺口或限制（NPU 無 silicon）。⛔ = M3／M6 是 Phase 2 才實作的整合層（設計使然，非 Phase 1 缺失）、M8 熱模型 pending。

一句話就緒判定：decode 主路徑（M1 CIM 算 + M2 記憶體搬 + M4-GPU attention + M4-CPU 支援 + M7 端到端 recompose）達量測級可信，多數單元 calibrated／ready；剩下三塊是誠實標註的缺口——M4-NPU 無 silicon（#13，simulated/borrowed，無裁判）、M8 熱模型 pending、多單元競爭（M3/M6）為 Phase 2 整合工作。沒有 blocking gap：所有就緒單元的介面已凍結、可直接接 Phase 2。
