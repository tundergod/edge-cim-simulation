Phase 1 總覽

這份站台把 Phase 1 的元件建模與驗證整理成自給自足的一份回顧——每個計算/記憶體單元現在多可信、缺什麼、能不能安全進入 Phase 2。核心紀律：不誇大、不軟化；沒量的就說沒量，每個數字都回查得到 committed artifact。

§0這個模擬器在做什麼

研究目標是一個以真實晶片校準的「CIM-enabled 異質行動 SoC 上 LLM 推論」模擬器。這種晶片並不存在，真實的 Axelera Metis 卡也無法直接當研究對象（Metis Alpha 跑不了完整 LLM、量產 Metis Card 只能透過封閉工具鏈跑）。

策略因此是：把研究對象設為一個模擬的 CIM-mobile SoC，再用兩塊真實 Metis 板（Aetina Metis Alpha + 量產 Metis Card）當校準 ground truth。完整動機、平台假設與貢獻定位見 OVERALL.md，此處不重述。

§1橋接假設 · 元件 → 端到端

整個專案的骨幹是 8 個模組 M1–M8 與 6 層驗證 L1–L6。整體論證的橋接假設是：把每個單元各自校準好的元件模型組合成端到端推論延遲，組合的結果仍貼合真實。

Phase 1 是「元件層」：把 Phase 0.3 量到的資料，對每個計算/記憶體單元擬合成方程式（而非龐大 lookup table），逐一驗證其準確度並誠實標註可信邊界。Phase 1 不做整合——把校好的元件串成端到端 event-driven 模擬器（M3 事件引擎 + M6 排程器）是 Phase 2 的工作。

因此本報告的結論不是「模擬器完成了」，而是「每個元件現在多可信、缺什麼、能不能安全進入 Phase 2」。橋接假設本身在 Phase 1 已有一個下界證據：端到端 decode recompose 在 8B 上的 hold-out 誤差為 {{recompose.err_8b_pct}}%（用 1B+3B 校準、外推 8B），證實組合可行；完整的逐 token 整機驗證仍屬 Phase 2。

§2方法論 · 為何按「資料來源」拆成三層

Phase 1 刻意依資料來源、而非依模組切分，因為一個元件可信到什麼程度，取決於它背後是哪一種證據：

Phase 1 三個資料來源層
子階段是什麼可信度定位
1.1 Metis silicon 校準凡能用自家 Metis 量測直接校準、並通過 ADR-0006 gate 的核心 decode 路徑：CIM 2D G_eff、Mali GPU attention、CPU 支援算子、量產卡 LPDDR4x、M5 trace，外加端到端 recompose 吞吐 hold-out（驗的是 tok/s，非能耗——M7 能耗無功耗遙測、屬 assumption 級）。量測級可信
1.2 模組化 analytic 層把每個非-micro-benchmark 單元改寫成「一個模型引擎 + 一份可換 spec 檔」，換型號＝換 spec。補齊 CPU 指令數 roofline、NPU 解析 systolic-roofline、全-spec 記憶體、GPU roofline 槽、CIM 雙拓樸與 Card 重驗。完整、可換型號
1.3 重型 sim drop-in把 ONNXim（NPU）、Ramulator2（記憶體）這類 C++ 重型 sim 透過凍結介面 engine= 插進 1.2 的同一個槽，交叉比對 analytic 的單串流趨勢。兩者都是 simulated、不是 silicon。交叉驗證（招牌價值留待 Phase 2）

三期共用一條凍結介面 Engine(spec).predict(workload) → {latency_us, bound, provenance}，所以 Phase 1.1 的校準在 1.2 改成 spec-based 後仍可重現（recompose 8B hold-out 重跑仍是 {{recompose.err_8b_pct}}%）。

§3誠實標籤 · 怎麼讀每個單元的 chip 列

每個單元頁的標題下方有一列誠實 chip，把該單元每種證據的可信度一眼標出。亮起（實心點、藍框）= 該單元有這種來源；劃掉（淡色刪除線）= 沒有這種來源——沒有，本身就是誠實呈現。橘框 = 亮起但有 caveat（如 assumption）。五個 chip 的固定含義：

calibrated fitted simulated assumption borrowed
誠實標籤定義（每個來源恰標一種）
標籤意義
calibrated對自家手上的真實 Metis silicon 擬合／驗證。
fitted參數化方程式擬合（可外推、可解釋、體積小），擬合誤差以 median / p95 相對誤差記錄。
simulated來自模型或外部模擬器（ONNXim / Ramulator2），未經自家 silicon 驗證。
assumption規格推算或文獻典型值，repo 內無一手資料來源（chip 染橘提醒）。
borrowed直接借用他人量測（主要是 HeteroInfer 的 trend）。
標籤不混用、可並存。一個元件可同時含多種來源（例如記憶體 = LPDDR4x calibrated + LPDDR5 simulated + 峰值 assumption），頁面會逐項分開標。chip 的 on 狀態與來源路徑都 committed 在 chips.json，audit 是對著這張表查、不是對著散文查。
§4怎麼讀這份報告 · 五段式單元模板

站台按單元編排（CIM / 記憶體 / CPU / GPU / NPU / Workload / 能量+E2E），對應 Phase 2 要組起來的異質 SoC 各計算單元。每個單元一律用同一個五段式模板：

每個單元頁的五段結構
段回答的問題
§0 本章角色這個單元模擬什麼最基本的算子、為什麼這幾件事就足夠。
§1 Measurement · 量測silicon 錨點從哪來、量了什麼標的。
§2 Fit · 擬合方程式形式 + 逐參數的物理意義（值由 JSON 注入）。
§3 Simulation · 模擬 / 外推重型 sim 交叉驗證，或量測 envelope 之外的解析外推。
§4 Validation gate · 驗證門檻誤差數字對 ADR-0006 gate 的 PASS / 待解判定。
§5 Phase-2 就緒度哪些介面凍結可直接調用、哪些是外推區或真缺口。

跨單元結果（M5 trace、M7 能耗、decode 端到端 recompose）收在後段；尚未實作的整合層（M3 / M6）明確區分「未實作（設計使然，是 Phase 2 工作）」與「真缺口」。每個數字皆從 committed JSON 注入，建置時遇到無法解析的 placeholder 會直接失敗——數字不能在散文裡被手打錯。完整的來源對照見「來源與誠實帳本」頁。
