來源與誠實帳本

站台裡每個量化數字都回查得到 committed artifact。本頁把各單元的主張對應到其來源檔案與欄位——驗證報告、擬合參數、可換 spec、原始量測。這些 artifact 是事實基準：散文若與之衝突，以 artifact 為準。

§0誠實紀律 · 數字怎麼進到頁面

站台的每個數字都從 committed JSON 注入，不在 HTML 裡手打。頁面原始碼帶 {{key}} placeholder；tools/report/_metrics.py 是這些 key 的唯一事實來源，逐一從 validation/reports/*.json / simulator/.../*.json 讀出並格式化。

build.py 在建置時替換所有 placeholder，並對任何未解析的 {{…}} 直接失敗（fail-loud）。所以一個數字不可能在散文裡被手打錯：它從 JSON 流出，JSON 一改頁面就跟著改；若來源欄位被移除，建置與 tests/test_report_metrics.py 會立刻報錯。每個單元頁的 chip 列（on 狀態 + 來源路徑）同樣 committed 在 chips.json，audit 對著表查、不對散文查。

來源紀律 · 每個數字皆回查 committed artifact
  1. 數字注入唯一來源：tools/report/_metrics.py（path + formatter，受 tests/test_report_metrics.py 守護）
  2. chip on 狀態 + 來源路徑：docs/report/phase1-site/chips.json
  3. 建置 fail-loud：docs/report/phase1-site/build.py（未解析 {{key}} 即 exit）
  4. 事實基準總覽：validation/reports/、simulator/{models/params,specs}/、measurements/、validation/contracts/、docs/adr/
